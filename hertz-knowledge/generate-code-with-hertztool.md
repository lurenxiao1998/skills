# Hertztool 代码生成工具

## 什么是 Hertztool?

Hertztool 是 Hertz 框架的官方代码生成工具,用于从 IDL (Thrift/Protobuf) 自动生成 HTTP 服务代码。

## 版本说明

### Hertztool v3 (推荐)

最新版本,解决了 v2 的历史问题,推荐使用。

**安装:**
```bash
go install code.byted.org/middleware/hertztool/v3@latest
```

**验证:**
```bash
hertztool -v
# 输出: hertztool version v3.1.5
```

### Hertztool v2 (已废弃)

不推荐使用,存在多个已知问题。

## 核心命令

### new - 创建新项目

```bash
hertztool new --psm=p.s.m --mod=code.byted.org/namespace/projectName
```

**参数:**
- `--psm`: 服务的 PSM 信息
- `--mod`: Go module 名称

**生成的目录结构:**
```
.
├── biz
│   ├── handler/        # Handler 实现
│   └── router/         # 路由注册
├── conf
│   └── hertz.config.yaml
├── main.go
├── router.go
└── script
    └── bootstrap.sh
```

### update - 更新/生成代码

从 IDL 文件生成或更新代码。

#### 基础用法

```bash
# Thrift
hertztool update -idl idl/api.thrift --mod=code.byted.org/namespace/projectName

# Protobuf
hertztool update -idl idl/api.proto --mod=code.byted.org/namespace/projectName
```

#### 常用参数

```bash
hertztool update \
  -idl idl/api.thrift \
  --mod=code.byted.org/namespace/projectName \
  --handler_dir=biz/handler \     # Handler 输出目录
  --model_dir=biz/model \         # Model 输出目录
  --router_dir=biz/router \       # Router 输出目录
  --client_dir=biz/client \       # Client 输出目录
  --base_domain=example.com       # 基础域名
```

### --insert 选项 (重要)

控制代码更新时的插入策略。

**不带 --insert:**
- 默认行为
- 只更新 model 和 router_gen.go (生成的路由代码)
- **不会修改** handler 和手动编写的路由注册代码
- 适合日常迭代,保护手写代码

**带 --insert:**
```bash
hertztool update -idl idl/api.thrift --mod=xxx --insert
```
- 会在 handler 和 router 中插入新的代码框架
- 适合新增接口时使用
- **注意:** 可能覆盖现有文件,需谨慎使用

**最佳实践:**
1. 首次生成使用 `--insert`
2. 后续更新不带 `--insert`,手动添加新 handler
3. 或者先备份再使用 `--insert`,然后 merge 代码

## IDL 文件管理

### 目录结构

```
project/
├── idl/
│   ├── api.thrift       # HTTP API 定义
│   ├── model.thrift     # 通用数据模型
│   └── common/
│       └── base.thrift  # 基础类型
├── biz/
└── main.go
```

### Thrift IDL 示例

```thrift
namespace go api

include "common/base.thrift"

struct GetUserRequest {
    1: required i64 id (api.query="id")
}

struct GetUserResponse {
    1: required base.User user
}

service UserService {
    GetUserResponse GetUser(1: GetUserRequest req) (api.get="/api/user")
}
```

### Protobuf IDL 示例

```protobuf
syntax = "proto3";

package api;

option go_package = "code.byted.org/namespace/projectName/biz/model/api";

message GetUserRequest {
    int64 id = 1; // @query
}

message GetUserResponse {
    User user = 1;
}

service UserService {
    rpc GetUser(GetUserRequest) returns (GetUserResponse) {
        option (api.get) = "/api/user";
    }
}
```

## 生成代码说明

### Handler 代码

生成在 `biz/handler/` 目录:

```go
package handler

import (
    "context"
    "code.byted.org/middleware/hertz/pkg/app"
    "code.byted.org/namespace/projectName/biz/model/api"
)

// GetUser .
// @router /api/user [GET]
func GetUser(ctx context.Context, c *app.RequestContext) {
    var req api.GetUserRequest
    err := c.BindAndValidate(&req)
    if err != nil {
        c.JSON(400, map[string]string{"error": err.Error()})
        return
    }

    // TODO: 实现业务逻辑
    resp := &api.GetUserResponse{}
    c.JSON(200, resp)
}
```

### 路由注册代码

生成在 `biz/router/register.go`:

```go
package router

import (
    "code.byted.org/middleware/hertz/byted"
    "code.byted.org/namespace/projectName/biz/handler"
)

func Register(h *byted.Engine) {
    customizedRegister(h)
}

// 自定义路由注册 (手动编辑)
func customizedRegister(h *byted.Engine) {
    h.GET("/api/user", handler.GetUser)
}
```

生成的 `router_gen.go` (自动生成,不要手动修改):

```go
// Code generated by hertztool. DO NOT EDIT.
package router

import (
    "code.byted.org/middleware/hertz/byted"
    "code.byted.org/namespace/projectName/biz/handler"
)

func GeneratedRegister(h *byted.Engine) {
    h.GET("/api/user", handler.GetUser)
}
```

## 常见问题

### 1. use of internal package not allowed

**错误信息:**
```
use of internal package github.com/andeya/goutil/internal/ameda not allowed
```

**解决方案:**
升级 Hertztool 到 v3 版本。

### 2. protoc 版本过低

**错误信息:**
```
invalid protoc version libprotoc 2.x, need protoc 3.x
```

**解决方案:**
安装 protoc 3.x 版本 (参考安装文档)。

### 3. 生成代码路径错误

**问题:** 生成的代码不在期望的目录。

**解决方案:**
使用 `--handler_dir`, `--model_dir`, `--router_dir` 明确指定输出目录。

### 4. IDL 更新后代码未变化

**原因:** 没有使用 `--insert` 选项。

**解决方案:**
- 使用 `hertztool update -idl xxx --insert`
- 或手动添加新生成的 handler/router 代码

## 最佳实践

### 1. 版本控制

- 将 IDL 文件纳入 Git 版本管理
- 生成代码也纳入版本管理,便于 Code Review

### 2. IDL 组织

- 按业务模块拆分 IDL 文件
- 使用 include/import 复用公共类型

### 3. 代码生成策略

- 首次生成使用 `--insert`
- 后续迭代不带 `--insert`,保护手写代码
- 重大重构前先备份

### 4. 注释规范

- 在 IDL 中添加详细注释
- 注释会被生成到 Go 代码中

### 5. 测试

- 生成代码后立即运行测试
- 确保 IDL 变更不破坏现有功能
