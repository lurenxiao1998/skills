# 工具和最佳实践

本文档详细介绍了 TikTok API 开发全流程中使用的工具平台和最佳实践，涵盖设计、原型、开发测试和发布四个阶段。

## 设计阶段工具

- **Lark Docs**：文档协作平台，用于 API 接口描述和团队沟通
- **行业参考工具**：Stoplight、ApiFox、Swagger Editor 等提供设计参考

### IDL/API 管理平台
- **DevFlow 平台**：基于 BAM 的 API 工作流程平台，推荐作为未来 API 规范实施和管理的一站式平台

### 设计规范与最佳实践

- **BAM PB IDL 定义规范**：TikTok 内部的 IDL 设计标准
- **接口管理平台最佳实践**：提供平台使用的最佳实践指导

## DevFlow 平台介绍

### 平台背景
DevFlow 平台从移动网关（Janus Mobile）发展而来，旨在解决 API 研发流程中的问题：
- **API 重要性**：API 是客户端与服务端交互的重要手段，大量业务需求涉及 API 字段变更
- **事故频发**：IDL 定义导致的事故反复出现，主要表现为服务端和客户端对接口定义不一致
- **流程复杂**：围绕 API 的研发流程有很多反复的机械步骤，流程长、步骤多
- **协作困难**：API 涉及多个角色协作，不同领域知识需要传递

### 平台功能
DevFlow 提供业务方可定制的协作流程，优化端到端的 API 交付效率和质量：

#### IDL 生命周期管理
- **IDL 在线编辑**：在线协作编辑，通知需求相关方参与设计 review
- **兼容性检测**：提供兼容性检测和规范检测
- **Review 管理**：Reviewer 维度可以到服务粒度配置
- **合入发布**：管理 IDL 的合入和发布流程

#### 多端代码生成
- **服务端 codegen**：生成服务端代码
- **客户端 codegen**：生成客户端代码
- **前端 codegen**：生成前端代码
- **字段裁剪**：支持字段裁剪功能

#### 标准协作流程
- **需求维度管理**：以需求维度管理 IDL 变更
- **托管协作环境**：降低上手成本
- **业务定制**：业务方可按需求定制协作流程，自由编排、扩展各个步骤

### 业务实践案例

#### 麦哲伦项目（Fanno, TikTok Shop）
**挑战**：新业务项目希望在研发初期通过定义完善的 API 研发流程，避免手写 API 不一致导致的事故

**解决方案**：
- Android、iOS、前端与服务端团队使用同一个 IDL 仓库做代码生成
- 服务端使用平台发起 IDL 变更，并触发各端 Codegen
- 客户端收到代码生成通知后使用版本号集成各端业务项目
- 客户端业务合码前卡点检查对应 IDL 任务已经合入主干

**收益**：
- 客户端 RD 调用接口类似 RPC 调用，不需要感知 HTTP 细节
- 没有出现过字段不一致的问题
- 标准化的 API 层在监控治理、协议升级改造等方面提供方便切面

#### 抖音某新 APP 项目
**问题**：
- 所有 API 都希望有 IDL 约束，IDL 修改后能及时通知到各端
- IDL 需要有业务自定义规范
- 研发节点太多，一次 IDL 变更需要手动操作的步骤太多
- pb_builder 依赖特定 protoc 版本，只能在 devbox 执行

**解决方案**：
- 建立独立的 IDL 仓库，三端共同使用
- 收敛仓库权限，所有 API 变更都从 mobile 平台发起
- 配置三端 Codegen 能力，IDL 变更后自动触发各端 codegen
- 平台提供 IDL 兼容性检测、冲突检测等

**收益**：
- 提供完善的 API 管控能力，字段变更和需求可以对应
- 保证各端代码和 IDL 严格一致
- 提升各端研发和协作效率
- 提高研发流程的规范性

## 最佳实践总结

### 设计阶段最佳实践
1. **使用 ProtoBuffer 定义 IDL**：PB 是行业认可的协议，具有良好的扩展性和表达能力
2. **RPC 协议使用 Thrift**：在大多数场景中，thrift 用作服务间进行 RPC 调用的协议和工具
3. **遵循 BAM PB IDL 定义规范**：确保 API 设计符合 TikTok 内部标准

### 开发阶段最佳实践
1. **服务端代码生成**：使用 PBBuilder 或 GDP Tool 生成服务器端模型和代码
2. **客户端代码生成**：使用基于 ByteX 的 BeanCopy 插件生成客户端代码
3. **API 测试**：使用 Rhino 进行压力测试，Tesla 平台进行自动化测试

### 协作流程最佳实践
1. **使用 DevFlow 平台**：作为 API 规范实施和管理的一站式平台
2. **标准化流程**：从文档约束转变为严格的流程约束
3. **自动化检查**：在联调阶段之前发现兼容性错误、语法错误等

### 监控治理最佳实践
1. **使用 Grafana 监控**：构建全面的 API 监控系统
2. **定期字段裁剪**：使用 BAM 字段裁剪功能清理无用字段
3. **服务优先级管理**：实施过载保护和服务优先级策略
