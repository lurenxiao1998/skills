# veImageX 错误码处理指南

veImageX 服务端 SDK 在调用 API 时可能会返回各种错误码，开发者需要根据错误码进行相应的错误处理。以下是主要的错误码分类和处理建议。

## 错误码分类

### 1. 公共错误码
veImageX 使用火山引擎的公共错误码体系，这些错误码适用于所有 API 接口。公共错误码主要包含：
- 认证鉴权相关错误
- 请求参数格式错误
- 服务端内部错误
- 网络超时错误

### 2. veImageX 特有错误码
veImageX 有自己的一套错误码体系，主要分为以下几类：

#### 图片处理相关错误码
- **5000**: 解码器构造失败，可能是图片格式不支持解码，或原图损坏无法解码
- **5001**: 解码失败，可能是图片格式不支持解码
- **5002**: 编码器构造失败，可能是输出格式不支持
- **5003**: 编码失败
- **5004**: 解密失败
- **5005**: 输入图片超过最大处理限制
- **5006**: 动图处理失败（不会返回该错误码，动图处理失败会降级为原图）
- **5007**: 动图抽帧失败
- **5008**: 静图不能转为动图
- **5009**: 图片处理 Filter 构建失败，或使用暗水印/文字水印功能时使用了非法字符
- **5010**: 图片处理失败，一般是 RPC 错误

#### 附加组件相关错误码
- **600003**: 输入参数错误
- **600011**: 算法处理失败
- **600012**: 图片解码失败
- **611400**: 图片分辨率越界

## 错误处理建议

### 1. 参数验证错误
- 检查请求参数是否符合 API 文档要求
- 验证图片格式、大小、分辨率是否在支持范围内
- 确认签名和认证信息是否正确

### 2. 图片处理错误
- 对于解码失败（5000、5001），检查原图是否损坏或格式是否支持
- 对于编码失败（5002、5003），确认输出格式是否支持
- 对于处理限制错误（5005），检查图片是否超过最大处理限制

### 3. 网络和服务错误
- 实现重试机制，特别是对于网络超时错误
- 监控服务可用性，关注服务不可用时间定义
- 设置合理的超时时间，避免长时间等待

### 4. 异步任务错误处理
对于异步任务（如多文件压缩、AI 图像处理），需要：
- 定期查询任务状态
- 处理任务失败时的错误码和错误信息
- 实现任务重试或补偿机制

## 最佳实践

1. **错误日志记录**：详细记录错误码、错误信息和相关上下文
2. **用户友好提示**：将技术错误码转换为用户可理解的信息
3. **监控告警**：对关键错误码设置监控告警
4. **降级处理**：对于非关键功能，实现优雅降级
5. **重试策略**：实现带退避算法的重试机制
