# Libra 配置空间管理指南

配置空间（Namespace）是 Libra 配置中心的核心管理单位，用于集中管理服务端配置参数和功能开关。每个配置空间对应一个业务或服务的配置集合，是配置下发的最小单位。

## 配置空间概念

**配置空间**是某业务/服务下所有相关配置的一个统一管理单位，概念上等同于顶层命名空间（Namespace）。在一期的功能实现上，配置空间是配置下发的最小单位。

### 配置空间基本信息
- **名称**：配置空间的唯一标识
- **描述**：对配置空间用途的简要说明
- **关联PSM**：关联 PSM 后，配置空间中的配置项将会下发到 PSM 所在的机器上
- **管理员**：可以编辑配置空间的信息，可以在配置空间下新增、修改和发布配置
- **通知Lark群**：绑定后，该 Lark 群可接收发布通知、报警等消息，若不配置则默认发送给所有管理员

## 配置空间类型

### 1. 基础版配置空间
适用于大多数服务端配置管理场景，支持：
- 配置的强 schema 校验
- 支持基础配置和差异化配置（比如新老用户某个配置的 value 可以不同）
- 配置快速修改、回滚和灰度发布，支持物理小流量发布
- 配置监控（配置在各机器上的生效版本，差异情况，业务方是否真的获取到配置等监控）

### 2. 升级版配置空间（条件配置管理）
适用于需要多维度条件取值的复杂场景，特点包括：
- 判断条件有固定层级
- key 的默认值逻辑按照条件层级继承
- 支持带有层级优先级的条件配置管理
- 在灰度 AB 实验环节支持条件化配置下发

**升级方式**：
- 新建配置空间：打开「判断条件有优先级」的开关
- 编辑配置空间：当且仅当配置空间下没有配置时可以更改

## 配置项管理

### 配置项类型
1. **子配置**：没有实际 value 的 key，其下常常嵌套子配置、值配置，功能类似于 namespace（建议控制在 3 层之内）
2. **值类型配置**：对应有 value 的 key，支持多种取值类型：
   - 整型、浮点型、字符串、布尔型
   - 字典、整形数组、浮点型数组、字符串数组、json
   - 支持按条件取值（if-else 逻辑）

### 配置项操作
- **增**：支持配置空间管理员新增配置项
- **删**：支持配置空间管理员删除配置项，子配置当且仅当其下没有任何配置项时才可删除
- **改**：支持配置空间管理员修改配置项
- **查**：支持通过搜索 key 或描述信息快速找到配置项

## 新版配置发布（2025年更新）

### 新旧版本对比
**配置发布（新版）**：
- 支持服务端、客户端的配置搜索：任何从实验全量到配置中心的 key 都可以在这里搜到
- 右上角直接创建新配置发布（无需通过实验）
- 新交互 & 性能更快 ⚡️

**特殊配置空间（旧版服务端）**：
- 非 Launch 类空间使用：本地固化类（例如 ad_predict 空间）、小包类
- 是旧版服务端配置空间中，非 Launch 类空间的使用类型

### 新版配置列表特性
1. **界面优化**：客户端、服务端合并到「配置发布」tab 下，输入配置 key / 配置名称 / 配置 ID / 取值 ID（VID）等进行搜索
2. **树状图切换**：点击"树状图"可切换成旧版树状视角
3. **关联实验查看**：清晰展示每个 key 关联的所有实验链接
4. **状态可视化**：
   - 全量状态&取值：当前配置是否全量，如已全量，鼠标 hover 在后方的代码框，可查下发逻辑
   - 最新版本状态&取值：是否存在未发布的版本
5. **多端生效展示**：以配置为粒度，对于同个 Key 生效在多个 App 一目了然

## 配置发布流程

### 从实验全量发布
**适用实验类型**：
- 服务端实验 ✅
- 普通客户端实验（需根据产品类型判断）
- AB 客户端 SDK ❌
- Web 端 ❌
- SettingsSDK ❌

**发布步骤**：
1. **快速全量发布入口**：
   - 实验详情右上角
   - 实验版本右上角
2. **选择全量实验版本**：
   - 原实验链接支持跳转
   - 选择全量的实验版本，如原实验包含特定 PSM，黄字提示全量后将对全局生效
   - 自动复制实验的取值条件（受众规则），可按需调整
3. **确认发布 key 及下发逻辑**：
   - 展示全量的 key，如有多个 key 将在顶部并列展示，需逐个确认
   - 配置取值展示 key 的多个 value（变体），允许不同下发逻辑
4. **发布确认**：
   - 确认与实验&配置的冲突
   - 全量后（100%），是否自动关闭原实验？
   - Diff 对比更清晰（上下模式/左右模式）

### 直接创建配置发布
**创建流程**：
1. **入口**：配置发布 tab 下的统一入口
2. **填写配置详情**：
   - 配置 key：支持字母、数字、下划线、小数点，暂不支持斜杠 /（预期 2025 Q3 进入排期）
   - 配置空间：选择 Launch_xxx 空间
   - 参数路径：选择完配置空间后展示，按需选择文件夹
3. **确认生效 App**：可一次性选多个 App/Token，分别配置下发逻辑
4. **保存并发布**：进入配置详情页生成新的版本

## 配置管理操作

### 查看当前配置
- 顶部一栏展示 key 的属性，右上角支持查看操作日志
- 编辑属性添加配置 owner
- 左上角支持切换不同 App/Token 查看
- 左列可看到所有的发布版本，以及每个版本的状态

### 重新发布配置
**重要概念**：服务端配置"回滚"不存在！
- 如有"未发布"版本，可在左列选择后发布
- 如没有合适版本，右上角"创建新版本"发布，从而覆盖当前下发
- 如彻底不要这个配置下发、也不要这个配置了，需要删除配置

### 删除配置
- "删除配置"相当于配置中心不再下发这个配置，且这个配置消失在配置中心
- 如"启用状态"开启，说明配置在线上仍生效，需先关停
- 点击关闭"启用状态"后，会触发下线流程
- "启用状态"关闭后，会直接删除配置，删除后无法恢复

## 权限管理

### 配置授权
- 权限以 key 粒度整合，可点击他人 key 页面的"权限申请"进行审批
- 如原配置 owner 已离职，平台管理员会兜底，进行审批
- 国内机房：需配置 owner 点击审批
- 海外机房：由于合规原因无法一键通过，需要配置 owner 进入配置详情，从右上角"编辑属性"，手动添加申请人为 owner

### 常见权限问题
**"我已经是 key owner 了，为啥还是提示'无权限'？"**
- 大概率只有这个 App 的"产品只读"权限，没有"产品成员"权限
- 无法创建实验、发布配置
- 解决方案：去产品管理-权限管理-角色管理，申请"产品成员"权限

## 最佳实践建议

1. **命名规范**：配置 key 中不包含 "." 字符，或使用自定义分隔符
2. **层级控制**：子配置建议控制在 3 层之内
3. **权限管理**：及时更新配置 owner，避免权限真空
4. **发布验证**：接入完成后需进行接入校验，验证实验配置能正确返回
5. **监控告警**：配置 SDK 会上报配置的使用情况，监控系统会严格监控业务拿到的配置是否与预期一致
