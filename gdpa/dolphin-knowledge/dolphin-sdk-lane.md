# Dolphin SDK泳道配置指南

Dolphin支持规则组维度的泳道能力，可以将一个规则组的改动发布到泳道环境中进行功能验证。在泳道环境中同一个事件下的其他规则组则仍按照线上正式环境的内容生效。

## 泳道环境概念

**泳道环境**是安全生产下的概念。在开发完成之后需要上线，但直接在线上环境测试代码非常危险，所以需要非线上环境下进行测试。为了解决环境冲突的问题，可以建立多个环境。泳道可以将环境区分，测试时在各自泳道测试而不发生冲突。

**基准环境**是已上线的环境作为'基准环境'。

## 泳道创建与配置

### 创建泳道环境
在规则编辑界面可以点击「新增」来新建一个泳道环境，或者是切换到其他泳道。

泳道的环境名称需要和公司内部多环境平台中创建的泳道环境名称一致。

泳道一旦创建不支持修改名称。

## 泳道规则更新

在规则发生变动后可以点击「保存」将规则的内容直接部署到泳道环境中，并立即生效。如果编辑完成后不保存，会将内容临时保存在草稿中（无论有多少个泳道环境，同一个规则组只有一份草稿）。

## 泳道上线流程

### 提交上线工单
功能验证完成后可以直接点击提交审核，会将该泳道环境中的规则通过工单的方式提交到正式环境。提交之前会展示线上规则与当前提交的规则的内容diff，确认后可以按照全量发布或者是灰度发布提交上线工单。

![提交上线工单界面](https://p-tika-sg.tiktok-row.net/tos-alisg-i-tika-sg/51033904445a44fcb6576fed13e2faec~tplv-tika-image.image)

### 工单完成后删除环境
在提交上线工单的时候会有选项「工单完成后删除当前环境」供勾选。选中后在工单完成之后会自动删除关联的泳道环境。

## 特殊泳道配置

### 引擎升级泳道
对于Dolphin SDK引擎升级，支持在泳道进行部署，这个泳道做了特殊的配置，会把事件的配置mock掉，直接使用新引擎来执行。

**操作步骤**：
1. 申请`ppe_dolphin_engine_upgrade`泳道的权限
2. 将直接接入Dolphin SDK的服务部署到这个泳道进行测试

两方验证没问题之后可以进入下一个阶段，完成离线验证之后，Dolphin这里会停止下发离线数据采集配置，也就是不再采集业务数据。

## 泳道使用注意事项

1. **基准环境要求**：如果想要在泳道使用，基准环境必须存在规则，可以先用false发布一次基准环境，后续再在ppe上配置相应业务规则

2. **发布流程**：发布必须灰度，建议业务对应的QA申请管理员权限，由业务QA审核

3. **权限管理**：如需修改请申请读写权限，如需查看请申请读权限

## 泳道优势

1. **隔离测试**：泳道环境中的请求会执行对应的规则，与其他环境隔离
2. **快速验证**：规则内容直接部署到泳道环境中并立即生效
3. **安全上线**：通过工单方式提交到正式环境，支持全量和灰度发布

## 代码示例

```go
// 泳道环境配置示例
laneConfig := &dolphin.LaneConfig{
    Environment: "ppe_dolphin_engine_upgrade", // 泳道环境名称
    EventName:   "your_event_name",
    RuleGroup:   "your_rule_group",
}

// 初始化Dolphin客户端时指定泳道环境
client, err := dolphin.NewClient(
    dolphin.WithLaneConfig(laneConfig),
    dolphin.WithEvent("your_event_name"),
)
```

通过泳道配置，开发者可以在不影响线上环境的情况下，安全地进行规则变更测试和功能验证。
