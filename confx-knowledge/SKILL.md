# confx 知识库

confx 是字节跳动内部的 Go 多数据源配置加载基础库，提供统一的配置管理能力，支持从 TCC、本地文件、环境变量等多种数据源加载配置，让业务代码专注于业务逻辑，无需关心具体的配置加载细节。

## 快速入门

confx 提供了简洁的 API 和灵活的配置方式，支持静态配置和动态配置两种模式。通过结构体标签定义配置项，可以轻松实现配置的加载和管理。

**核心概念**：
- **静态配置**：初始化时一次性加载，适用于不经常变化的配置
- **动态配置**：支持定期刷新和变化回调，适用于需要实时更新的配置
- **Loader**：从数据源获取内容的具体实现
- **Parser**：解序列化的具体实现

**使用场景**：
- 从本地 yaml/json 文件加载数据库 PSM 等配置
- 从 TCC 远程加载可动态变化的开关状态配置
- 从 KMS 加载需要加密的秘钥配置
- 统一管理多环境、多数据源的配置

[quick-start.md](./quick-start.md) - 快速入门指南

## 配置源支持

confx 支持多种配置数据源，包括默认集成的 TCC、文件、环境变量，以及可通过扩展引入的 KMS、Flag 等。每种数据源都有特定的参数和用法，满足不同场景的需求。

**主要配置源**：
- **TCC**：支持 TCC V2/V3，提供动态配置刷新、PSM 指定、ConfSpace 等特性
- **文件**：支持本地文件配置加载，支持路径映射
- **环境变量**：支持从环境变量读取配置
- **Byteconf**：支持 Byteconf 配置源，包括决策树场景

**使用场景**：
- 需要从 TCC 读取动态开关配置
- 从本地配置文件加载服务配置
- 通过环境变量管理不同环境的配置
- 使用 Byteconf 实现复杂的配置决策逻辑

[config-sources.md](./config-sources.md) - 配置源详细说明

## 动态配置特性

confx 的动态配置以函数字段形式声明，支持多种调用模式和丰富的特性，包括默认值、错误处理、深拷贝、回调等，满足不同业务场景的需求。

**核心特性**：
- **三种函数模式**：无默认值无 error、有默认值无 error、无默认值有 error
- **运行时指定 Key**：支持在运行时动态指定配置 Key
- **深拷贝保护**：避免配置对象被意外修改
- **变化回调**：配置更新时触发业务回调
- **监控指标**：内置监控面板，可通过 Grafana 查看

**使用场景**：
- 需要配置热更新能力的业务场景
- 对配置安全性要求高的场景
- 需要监控配置加载状态的场景
- 配置变更需要触发业务逻辑的场景

## 高级功能与最佳实践

confx 提供了丰富的高级功能和优化建议，帮助开发者编写更安全、高效的配置代码。包括 Mock 功能、性能优化、错误排查等。

**高级功能**：
- **Mock 功能**：完整的 Mock 支持，方便测试中模拟配置
- **Options 配置**：丰富的初始化选项，如刷新间隔、回调、调试模式等
- **性能优化**：提供深拷贝、内存使用等优化建议
- **错误排查**：提供配置不符等常见问题的排查方法

**最佳实践**：
- 静态配置使用 `required` 关键字进行初始化校验
- 动态配置根据业务需要选择是否使用 `deepcopy`
- 大对象配置谨慎使用深拷贝以避免性能问题
- 合理配置 TCC 超时和重试参数保证稳定性

**使用场景**：
- 编写单元测试时需要 Mock 配置
- 需要优化配置加载性能的场景
- 排查配置相关问题的场景
- 编写生产级配置代码的场景
